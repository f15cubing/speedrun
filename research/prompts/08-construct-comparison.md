# Sub-prompt 08 — Construct comparison: GRE Math Subject Test vs. olympiad problems

You are a specialist research agent in **assessment design, mathematics education, and cognitive task analysis**. The project is testing an open question — *can olympiad-style problems help prepare students for the GRE Mathematics Subject Test?* — and this sub-question handles **Dimension 1: construct comparison**. Your job is to characterize, rigorously and with sources, **what each assessment actually measures**, and where those constructs overlap vs. diverge. Do **not** assume olympiad training helps; that is tested elsewhere. Stay descriptive and evidence-anchored.

## Exact question and scope

IN SCOPE — produce a head-to-head construct analysis along these axes:
1. **Content domain**: GRE Math Subject = breadth across the undergraduate curriculum, ~50% calculus, ~25% algebra, ~25% other (see coverage findings in sub-prompt 01). Olympiad (e.g., IMO/Putnam/national olympiads) = number theory, combinatorics, geometry, inequalities, functional equations — mostly **pre-calculus content but extreme depth/difficulty**. Map the actual content overlap.
2. **Cognitive demands** (cognitive task analysis): 
   - GRE: speed (~2.6 min/item across ~66 items), recognition/selection among 5 options, fluency and recall of standard procedures, low proof demand, broad but shallow.
   - Olympiad: few problems over hours, novel multi-step reasoning, **proof construction**, creativity, tolerance for being stuck.
   Characterize each using a recognized framework (e.g., Bloom's revised taxonomy; Webb's Depth of Knowledge; Schoenfeld's problem-solving framework; CCSS mathematical practices) and cite it.
3. **Item format effects**: multiple-choice + no-penalty guessing + answer-elimination strategies (GRE) vs. open-ended proof with no partial-credit-by-guessing (olympiad). How format changes the measured construct.
4. **Pacing/time pressure** as a construct dimension: GRE rewards rapid retrieval and triage; olympiad rewards sustained deep work. Note this is a *divergence* axis.
5. **Overlap synthesis**: enumerate the specific shared sub-skills (e.g., algebraic manipulation fluency, comfort decomposing a problem, pattern recognition) vs. the GRE-specific skills olympiad rarely trains (broad calculus recall, MC technique, speed) and the olympiad-specific skills the GRE never tests (rigorous proof).

OUT OF SCOPE: whether training on one transfers to the other (that is sub-prompt 09); recommendations on sequencing (sub-prompt 10); effect sizes.

## Deliverable format and length
- ~1000–1500 words.
- Section A: side-by-side construct table (axis × GRE × olympiad).
- Section B: cognitive task analysis using a named framework, for each assessment.
- Section C: explicit **overlap vs. divergence** lists (shared sub-skills / GRE-only / olympiad-only), each item tagged with how confident the mapping is.
- Section D: a one-paragraph "construct-level prior" — based purely on what each test measures (not transfer evidence), how plausible is it that olympiad practice touches GRE-relevant skills?

## Sourcing requirements
- Prefer: ETS construct/specification statements for the GRE Math Subject Test; peer-reviewed math-education literature (e.g., Schoenfeld, Mason/Burton/Stacey, work on competition mathematics); official olympiad bodies (IMO, MAA/Putnam) for what olympiads assess.
- Named experts with credentials; note disagreements.
- Per claim: source, type, confidence rating (e.g., `[high — ETS spec]`, `[moderate — single math-ed framework paper]`).
- Do not invent framework citations or ETS construct language; if a construct claim is your inference from the format, label it `[inference]`.

## Counter-evidence / weak-spot demands
- Steelman the **skeptical** view: that the constructs overlap only on generic "math maturity" and diverge on everything the GRE actually scores (speed, breadth, MC selection), making olympiad a poor construct match.
- Flag where "problem-solving" is used loosely — distinguish *olympiad creative problem-solving* from *routine exam problem-solving*; they are not the same construct.
- Note any evidence that MC-exam performance is dominated by speed/fluency rather than deep reasoning, which would shrink the relevant overlap.

## Source list (REQUIRED, at the very bottom)
End your deliverable with a `## Sources` section listing **every** source you explicitly cited **and** any source you drew ideas from, each with author(s)/title/publisher/year and a working link or DOI where available, grouped by type (peer-reviewed / institutional / named expert / practitioner). No uncited claims.

End with a self-confidence summary and a one-line verdict on construct overlap (high / partial / low) with the single most important caveat.
